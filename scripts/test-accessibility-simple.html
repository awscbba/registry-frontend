🔍 Browser Accessibility Testing Tool

Purpose: This tool helps verify WCAG 2.1 AA compliance for the People Registry frontend application.

Note: This is a manual testing guide. For automated testing, use the Playwright script.

📋 How to Use This Tool:

  1. Open your browser's DevTools (F12 or Cmd+Option+I)
  2. Navigate to the Lighthouse tab (in Chrome) or Accessibility tab (in Firefox)
  3. Run an accessibility audit
  4. Use the checklist below to manually verify additional requirements
  5. Document any issues found

🎯 Manual Accessibility Checklist

1. Color Contrast (WCAG 2.1 AA - 4.5:1 for normal text)

2. Keyboard Navigation

3. Focus Indicators

4. ARIA Attributes

5. Focus Management

6. Screen Reader Compatibility

7. Component-Specific Tests

🛠️ Browser DevTools Testing

Chrome Lighthouse

  1. Open Chrome DevTools (F12)
  2. Go to "Lighthouse" tab
  3. Select "Accessibility" category
  4. Click "Analyze page load"
  5. Review the accessibility score (should be 90+)
  6. Fix any issues reported

Firefox Accessibility Inspector

  1. Open Firefox DevTools (F12)
  2. Go to "Accessibility" tab
  3. Click "Check for issues"
  4. Review contrast, keyboard, and text label issues
  5. Use the accessibility tree to verify structure

axe DevTools Extension

  1. Install axe DevTools extension for Chrome/Firefox
  2. Open DevTools and go to "axe DevTools" tab
  3. Click "Scan ALL of my page"
  4. Review violations by severity
  5. Fix critical and serious issues first
